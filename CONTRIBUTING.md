# Contributing

Thanks for your interest in contributing.

## Development Setup
1. Clone the repo
2. Load unpacked extension from this root folder in `chrome://extensions`
3. Make changes and reload the extension

## Pull Request Checklist
- Keep scope focused
- Verify main flows on Overleaf (`www.overleaf.com` and `cn.overleaf.com` if possible)
- Run basic syntax checks if JS files changed (`node --check <file>`)
- Update README when behavior changes

## Commit Style
Use concise prefixes like:
- `feat:` new functionality
- `fix:` bug fix
- `chore:` maintenance/docs/refactor
